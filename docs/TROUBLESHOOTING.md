## UI Not Loading / 404 Errors

### Symptom
- Development server starts successfully
- Backend API responds correctly
- Frontend returns `HTTP 404 Not Found` when accessing `http://localhost:8045`
- Browser shows blank page or "Not Found"

### Root Cause
The `base_path` configuration in `frontend/Dioxus.toml` is set, causing the app to be served from a subdirectory instead of the root path.

### Solution
1. Open `frontend/Dioxus.toml`
2. Check the `[web.app]` section
3. **Remove** the `base_path` line entirely:

**INCORRECT (causes 404):**
```toml
[web.app]
title = "Alashore Marine Exports | Premium Seafood Exporter, India"
base_path = "alashore-marine-pvt-ltd"  # ❌ Remove this line for development
```

**CORRECT (for development):**
```toml
[web.app]
title = "Alashore Marine Exports | Premium Seafood Exporter, India"
# No base_path line ✅
```

4. Restart the dev server:
```bash
pnpm run kill-ports
pnpm dev
```

5. Access the app at `http://localhost:8045`

### When to Use `base_path`
Only set `base_path` for production GitHub Pages deployment where the app needs to be served from a repository subdirectory (e.g., `https://username.github.io/repo-name/`). For all local development and standard deployments, omit `base_path`.

## Development Server Issues

### Port Already in Use

**Symptom:** Error messages like "Address already in use" or server fails to start.

**Solution:**
```bash
# Kill processes on ports 8045 and 8046
pnpm run kill-ports

# Or manually:
lsof -ti:8045 | xargs kill -9
lsof -ti:8046 | xargs kill -9
```

### Frontend Build Hangs or Fails

**Symptom:** `dx serve` command hangs or fails with cryptic errors.

**Solution:**
```bash
# Clean build artifacts
cd frontend
cargo clean
rm -rf dist/
rm -rf target/

# Rebuild
cd ..
pnpm dev
```

### Hot Reload Not Working

**Symptom:** Changes to source files don't trigger rebuilds.

**Solution:**
1. Check `frontend/Dioxus.toml` has correct watch paths:
```toml
[web.watcher]
reload_html = true
watch_path  = ["src", "assets"]
```

2. Restart dev server with `pnpm dev`
3. If still not working, press `r` in the `dx serve` terminal to manually rebuild

## Build Errors

### Cargo Workspace Profile Warnings

**Symptom:**
```
warning: profiles for the non root package will be ignored
```

**Solution:** This is expected. Profile settings must be in the root `Cargo.toml`, not in member crates. This warning is harmless and can be ignored.

### Missing Dependencies

**Symptom:** Compile errors about missing crates.

**Solution:**
```bash
# Update dependencies
cargo update

# Clean and rebuild
cargo clean
cargo build
```

### Frontend Wasm Build Fails

**Symptom:** Errors related to `wasm32-unknown-unknown` target.

**Solution:**
```bash
# Ensure wasm target is installed
rustup target add wasm32-unknown-unknown

# Rebuild
cd frontend
dx build --release
```

## CSS/Styling Issues

### Styles Not Applying

**Symptom:** UI appears unstyled or CSS changes don't take effect.

**Solution:**
1. **Never edit `frontend/assets/css/main.css` directly** - it's generated by Tailwind
2. Edit `frontend/input.css` instead
3. Rebuild CSS:
```bash
pnpm run build:tailwind
```

4. For watch mode:
```bash
pnpm run build:tailwind:watch
```

### Missing CSS File in Browser

**Symptom:** Browser console shows 404 for `/assets/css/main.css`.

**Solution:**
```bash
# Ensure CSS is copied to public directory
cd frontend
tailwindcss -i ./input.css -o ./assets/css/main.css
cp ./assets/css/main.css ./public/assets/css/main.css
```

This is handled automatically by `pnpm dev`, but if you see this error, run the commands manually.

## Backend API Issues

### CORS Errors

**Symptom:** Browser console shows CORS policy errors when frontend calls backend.

**Solution:**
1. Check `ALLOWED_ORIGINS` environment variable includes your frontend URL:
```bash
export ALLOWED_ORIGINS="http://localhost:8045,http://127.0.0.1:8045"
```

2. Restart backend:
```bash
pnpm run dev:backend
```

### Database Connection Errors

**Symptom:** Backend fails to start with SQLite errors.

**Solution:**
```bash
# Remove corrupted database
rm -f backend/data/alashore.db

# Restart backend (will recreate and seed DB)
cargo run -p backend
```

### Rate Limiting Blocking Requests

**Symptom:** API returns `429 Too Many Requests`.

**Solution:**
1. Check `RATE_LIMIT_RPM` in backend environment
2. Increase limit for development:
```bash
export RATE_LIMIT_RPM=100
cargo run -p backend
```

3. Or temporarily disable rate limiting in `backend/src/main.rs` (for development only):
```rust
// Comment out the rate limiter layer
// .layer(middleware::from_fn(rate_limit::rate_limit))
```

## Deployment Issues

### GitHub Pages 404 After Deployment

**Symptom:** GitHub Pages shows 404 for all routes except homepage.

**Solution:**
1. Ensure `.nojekyll` file exists in deployment:
```bash
touch frontend/dist/.nojekyll
```

2. Verify `base_path` in `frontend/Dioxus.toml` matches repository name:
```toml
[web.app]
base_path = "your-repo-name"  # For GitHub Pages only
```

3. Ensure GitHub Pages is configured to deploy from the correct branch/folder in repository settings.

### Missing Assets After Build

**Symptom:** Production build doesn't include CSS, images, or other assets.

**Solution:**
1. Verify `frontend/public/` contains all required assets:
   - `public/assets/css/main.css`
   - `public/assets/images/*.webp`

2. Rebuild with:
```bash
pnpm run build:tailwind
cd frontend
dx build --release
```

3. Check `frontend/dist/` contains copied assets

### Docker Build Fails

**Symptom:** Docker build errors or container doesn't start.

**Solution:**
1. Verify Dockerfile multi-stage build has correct paths
2. Ensure Tailwind CSS is built before frontend build:
```dockerfile
# In Dockerfile
RUN cd frontend && tailwindcss -i ./input.css -o ./assets/css/main.css
RUN cd frontend && dx build --release
```

3. Check health endpoint is accessible:
```bash
docker exec -it <container> curl http://localhost:8046/health
```

## Quick Diagnostic Checklist

When troubleshooting any issue, run through this checklist:

### ✅ Development Environment
- [ ] Node.js and pnpm installed
- [ ] Rust toolchain up to date (`rustup update`)
- [ ] wasm32-unknown-unknown target installed
- [ ] Dioxus CLI installed (`cargo install dioxus-cli`)
- [ ] Tailwind CSS CLI available

### ✅ Configuration Files
- [ ] `frontend/Dioxus.toml` has no `base_path` (for dev)
- [ ] `frontend/input.css` exists
- [ ] `frontend/public/assets/css/main.css` exists
- [ ] Backend `.env` or environment variables set

### ✅ Running Processes
- [ ] No zombie processes on ports 8045/8046
- [ ] Backend running on port 8046
- [ ] Frontend running on port 8045
- [ ] Tailwind watcher running (optional)

### ✅ Build Artifacts
- [ ] `target/` directory exists
- [ ] `frontend/dist/` directory exists (after build)
- [ ] No stale `.lock` files causing issues

## Getting Help

If you've tried the solutions above and still have issues:

1. **Check Recent Changes:**
   ```bash
   git status
   git diff
   ```
   Look for accidental configuration changes.

2. **Check Logs:**
   - Backend logs show database, server, and request info
   - Frontend logs in browser console (F12)
   - `dx serve` terminal output shows build errors

3. **Clean Slate:**
   ```bash
   # Nuclear option - clean everything and rebuild
   pnpm run kill-ports
   cargo clean
   rm -rf frontend/dist/
   rm -rf target/
   pnpm install
   pnpm dev
   ```

4. **Test Components Individually:**
   ```bash
   # Test backend only
   cargo run -p backend
   curl http://localhost:8046/health

   # Test frontend only (backend must be running)
   cd frontend
   dx serve --port 8045
   ```

## Prevention Best Practices

To avoid these issues in the future:

1. **Always use `pnpm dev`** for development - it handles server startup, port cleanup, and CSS building
2. **Never commit `base_path` in development** - only add it for deployment branches
3. **Edit `input.css`, not `main.css`** - the latter is generated
4. **Run tests before committing:** `cargo test --workspace && cargo clippy --workspace`
5. **Keep dependencies updated:** `cargo update && pnpm update`
6. **Review `CLAUDE.md`** before making architectural changes
7. **Check `docs/CURRENT_STATUS.md`** for latest known issues
